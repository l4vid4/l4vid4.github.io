2. JVMеҶ…еӯҳеҲҶе“ӘеҮ дёӘеҢәпјҢжҜҸдёӘеҢәзҡ„дҪңз”ЁжҳҜд»Җд№Ҳ

еҗҚз§°жҳҜеҗҰзәҝзЁӢз§Ғжңүдё»иҰҒдҪңз”ЁиҜҙжҳҺ
ж–№жі•еҢәпјҲMethod AreaпјүеҗҰпјҲзәҝзЁӢе…ұдә«пјүеӯҳж”ҫзұ»зҡ„е…ғдҝЎжҒҜпјҲзұ»з»“жһ„гҖҒеёёйҮҸжұ гҖҒйқҷжҖҒеҸҳйҮҸгҖҒJIT зј–иҜ‘д»Јз ҒзӯүпјүгҖӮJava 8 еҗҺдёәе…ғз©әй—ҙпјҢдҪҝз”Ёжң¬ең°еҶ…еӯҳгҖӮ
е ҶеҶ…еӯҳпјҲHeapпјүеҗҰпјҲзәҝзЁӢе…ұдә«пјүеӯҳеӮЁжүҖжңүеҜ№иұЎе®һдҫӢе’Ңж•°з»„пјҢжҳҜ GC зҡ„дё»иҰҒеҢәеҹҹгҖӮ
Java ж ҲеҶ…еӯҳпјҲJVM StackпјүжҳҜпјҲзәҝзЁӢз§Ғжңүпјүж–№жі•и°ғз”Ёж—¶еҲӣе»әж Ҳеё§пјҢдҝқеӯҳеұҖйғЁеҸҳйҮҸгҖҒж“ҚдҪңж•°ж ҲгҖҒиҝ”еӣһеҖјзӯүгҖӮ
Native ж ҲеҶ…еӯҳпјҲNative Method StackпјүжҳҜпјҲзәҝзЁӢз§Ғжңүпјүз”ЁдәҺж”ҜжҢҒ Native ж–№жі•зҡ„и°ғз”ЁпјҲеҰӮ JNIпјүгҖӮ
зЁӢеәҸи®Ўж•°еҷЁпјҲProgram CounterпјүжҳҜпјҲзәҝзЁӢз§ҒжңүпјүжҢҮзӨәеҪ“еүҚзәҝзЁӢжүҖжү§иЎҢзҡ„еӯ—иҠӮз ҒиЎҢеҸ·ең°еқҖпјҢзұ»дјјвҖңиЎҢеҸ·жҢҮй’ҲвҖқгҖӮ

3. Javaдёӯеһғеңҫеӣһж”¶пјҲGCпјүзҡ„ж–№ејҸжңүе“Әдәӣ

жҢүеӣһж”¶з®—жі•еҲҶзұ»

з®—жі•еҗҚз§°еҺҹзҗҶз®Җд»ӢдјҳзӮ№зјәзӮ№йҖӮз”ЁеңәжҷҜ
ж Үи®°-жё…йҷӨ (Mark-Sweep)ж Үи®°жүҖжңүеӯҳжҙ»еҜ№иұЎ вҶ’ жё…йҷӨжңӘж Үи®°еҜ№иұЎе®һзҺ°з®ҖеҚ•пјҢж— йңҖеӨҚеҲ¶пјҢиҠӮзңҒз©әй—ҙдјҡдә§з”ҹеҶ…еӯҳзўҺзүҮпјҢеӣһж”¶йҖҹеәҰж…ўпјҢеҲҶй…ҚеӨ§еҜ№иұЎеҸҜиғҪеӨұиҙҘиҖҒе№ҙд»ЈпјҲж—©жңҹдҪҝз”Ёпјү
еӨҚеҲ¶з®—жі• (Copying)еӯҳжҙ»еҜ№иұЎеӨҚеҲ¶еҲ°еҸҰдёҖеқ—еҢәеҹҹпјҢжё…з©әеҺҹеҢәеҹҹж— зўҺзүҮпјҢеҲҶй…Қиҝһз»ӯеҶ…еӯҳпјҢеӣһж”¶еҝ«з©әй—ҙжөӘиҙ№еӨ§пјҲеҸӘиғҪз”ЁдёҖеҚҠпјүпјҢеҜ№иұЎеӯҳжҙ»зҺҮй«ҳж—¶пјҢеӨҚеҲ¶ејҖй”ҖеӨ§ж–°з”ҹд»ЈпјҲеҜ№иұЎз”ҹе‘Ҫе‘Ёжңҹзҹӯпјү
ж Үи®°-ж•ҙзҗҶ (Mark-Compact)ж Үи®°еӯҳжҙ»еҜ№иұЎ вҶ’ ж•ҙзҗҶеҲ°дёҖз«Ҝ вҶ’ жё…зҗҶж— ж•ҲеҜ№иұЎж— зўҺзүҮпјҢдёҚжөӘиҙ№еҶ…еӯҳпјҢж”ҜжҢҒеӨ§еҜ№иұЎеҲҶй…Қз§»еҠЁеҜ№иұЎжҲҗжң¬й«ҳпјҢж•ҙзҗҶиҖ—ж—¶иҖҒе№ҙд»ЈпјҲеҜ№иұЎеӯҳжҙ»зҺҮй«ҳпјү
еҲҶд»Јж”¶йӣҶ (Generational)е°ҶеҶ…еӯҳеҲ’еҲҶдёәж–°з”ҹд»ЈгҖҒиҖҒе№ҙд»ЈпјҢеҲҶеҲ«дҪҝз”ЁдёҚеҗҢз®—жі•пјҲеҰӮеӨҚеҲ¶ + ж Үи®°ж•ҙзҗҶпјүз»јеҗҲеӨҡз§Қз®—жі•дјҳзӮ№пјҢй«ҳж•ҲйҖӮеә”дёҚеҗҢз”ҹе‘Ҫе‘Ёжңҹзҡ„еҜ№иұЎе®һзҺ°еӨҚжқӮпјҢеҲҶд»Ји°ғж•ҙзӯ–з•ҘйңҖиҰҒи°ғдјҳJVM й»ҳи®Өзӯ–з•Ҙ

жҢүеһғеңҫеӣһж”¶еҷЁеҗҚз§°еҲҶзұ»пјҲHotSpot JVM еёёи§Ғ GCпјү

еӣһж”¶еҷЁйҖӮз”ЁеҢәеҹҹзү№зӮ№
Serialж–°з”ҹд»ЈпјҲеӨҚеҲ¶з®—жі•пјүеҚ•зәҝзЁӢпјҢйҖӮз”ЁдәҺе°ҸеҶ…еӯҳгҖҒеҚ•ж ёгҖӮ
ParNewж–°з”ҹд»ЈпјҲеӨҚеҲ¶з®—жі•пјүеӨҡзәҝзЁӢзүҲ SerialпјҢеҸҜдёҺ CMS жҗӯй…ҚдҪҝз”ЁгҖӮ
Parallel ScavengeпјҲеҗһеҗҗйҮҸдјҳе…Ҳпјүж–°з”ҹд»ЈпјҲеӨҚеҲ¶з®—жі•пјүеӨҡзәҝзЁӢпјҢиҝҪжұӮй«ҳеҗһеҗҗйҮҸпјҢй…ҚеҗҲ Parallel Old дҪҝз”ЁгҖӮ
Parallel OldиҖҒе№ҙд»ЈпјҲж Үи®°-ж•ҙзҗҶпјүParallel Scavenge зҡ„иҖҒе№ҙд»Јеӣһж”¶еҷЁгҖӮ
CMSпјҲConcurrent Mark SweepпјүиҖҒе№ҙд»ЈпјҲж Үи®°-жё…йҷӨпјүе№¶еҸ‘дҪҺеҒңйЎҝпјҢйҖӮеҗҲе“Қеә”ж—¶й—ҙж•Ҹж„ҹеә”з”ЁпјӣжңүзўҺзүҮй—®йўҳгҖӮ
G1пјҲGarbage Firstпјүж–°з”ҹд»Ј+иҖҒе№ҙд»ЈпјҲеҲҶеҢәеӣһж”¶пјүж”ҜжҢҒе№¶еҸ‘гҖҒдҪҺеҒңйЎҝгҖҒеҸҜйў„жөӢпјҢйҖӮеҗҲеӨ§еҶ…еӯҳгҖӮ
ZGCпјҲJava 11+пјүе…ЁеҢәеҹҹе№¶еҸ‘еӣһж”¶жһҒдҪҺеҒңйЎҝпјҲ<10msпјүпјҢж”ҜжҢҒеӨ§еҶ…еӯҳпјҲTB зә§пјүгҖӮ
ShenandoahпјҲJava 12+пјүе…ЁеҢәеҹҹе№¶еҸ‘еӣһж”¶зұ»дјј ZGCпјҢйҖӮеҗҲдҪҺе»¶иҝҹеңәжҷҜгҖӮ

4. еҰӮдҪ•еҲӨж–ӯдёҖдёӘеҜ№иұЎжҳҜеҗҰеӯҳжҙ»(жҲ–иҖ…GCеҜ№иұЎзҡ„еҲӨе®ҡж–№жі•)

еј•з”Ёи®Ўж•°жі•пјҲReference CountingпјҢе·Іж·ҳжұ°пјү

еҺҹзҗҶпјҡ

  • жҜҸдёӘеҜ№иұЎз»ҙжҠӨдёҖдёӘеј•з”Ёи®Ўж•°еҷЁпјҢиў«еј•з”ЁдёҖж¬Ўи®Ўж•°еҠ дёҖпјҢеј•з”Ёж–ӯејҖеҮҸдёҖпјӣ
  • и®Ўж•°дёә 0 зҡ„еҜ№иұЎдјҡиў«еҲӨе®ҡдёәеһғеңҫгҖӮ

зјәзӮ№пјҡ

  • ж— жі•еӨ„зҗҶеҫӘзҺҜеј•з”ЁпјҢжҜ”еҰӮдёӨдёӘеҜ№иұЎдә’зӣёеј•з”ЁдҪҶе·Іж— еӨ–йғЁеј•з”ЁпјҢд»Қз„¶ж— жі•иў«еӣһж”¶пјӣ
  • еӣ жӯӨиў« HotSpot JVM зӯүдё»жөҒиҷҡжӢҹжңәж·ҳжұ°гҖӮ

еҸҜиҫҫжҖ§еҲҶжһҗз®—жі•пјҲReachability AnalysisпјҢдё»жөҒпјү

еҺҹзҗҶпјҡ

  • д»ҺдёҖз»„з§°дёә GC Roots зҡ„еҜ№иұЎеҮәеҸ‘пјҢеҗ‘дёӢжҗңзҙўпјҢиғҪиҝһйҖҡзҡ„еҜ№иұЎдёәеҸҜиҫҫеҜ№иұЎпјҢеҗҰеҲҷдёәеһғеңҫпјӣ
  • дёҚеҸҜиҫҫ вү  з«ӢеҚіжӯ»дәЎпјҢиҝҳеҸҜиғҪиҝӣе…Ҙ вҖңеӣӣж¬ЎеҲӨжӯ»вҖқ жңәеҲ¶гҖӮ

GC Roots еҢ…жӢ¬пјҡ

  • иҷҡжӢҹжңәж Ҳдёӯзҡ„еј•з”ЁеҸҳйҮҸпјҲж–№жі•ж Ҳеё§дёӯзҡ„еұҖйғЁеҸҳйҮҸиЎЁпјү
  • ж–№жі•еҢәдёӯзҡ„зұ»йқҷжҖҒеұһжҖ§
  • ж–№жі•еҢәдёӯзҡ„еёёйҮҸеј•з”Ё
  • жң¬ең°ж–№жі•ж Ҳдёӯ JNI еј•з”ЁпјҲNative ж–№жі•пјү

еҜ№иұЎвҖңзңҹжӯЈжӯ»дәЎвҖқеүҚзҡ„еҲӨж–ӯиҝҮзЁӢ

  • еҜ№дәҺдёҚеҸҜиҫҫеҜ№иұЎпјҢиҝҳиҰҒз»ҸеҺҶеҰӮдёӢеҲӨе®ҡжүҚдјҡиў«еҪ»еә•GCпјҡ
еҲӨе®ҡйҳ¶ж®өжқЎд»¶иҜҙжҳҺ
з¬¬дёҖж¬ЎеҲӨе®ҡжҳҜеҗҰйҮҚеҶҷ finalize() ж–№жі•еҰӮжһңжІЎжңүпјҢз«ӢеҚіеҲӨе®ҡдёәеһғеңҫ
з¬¬дәҢж¬ЎеҲӨе®ҡжү§иЎҢ finalize()пјҢжҳҜеҗҰйҮҚж–°е»әз«Ӣеј•з”Ёй“ҫжңүеј•з”ЁеҲҷвҖңеӨҚжҙ»вҖқпјӣеҗҰеҲҷиҝӣе…ҘдёӢдёҖиҪ® GC
жңҖз»Ҳеӣһж”¶finalize еҗҺд»ҚдёҚеҸҜиҫҫеҲӨе®ҡжӯ»дәЎпјҢGC еӣһж”¶

finalize()пјҡ

  • еңЁеҜ№иұЎиў« GC еүҚеҒҡдёҖдәӣиө„жәҗйҮҠж”ҫгҖҒж—Ҙеҝ—и®°еҪ•зӯүжё…зҗҶе·ҘдҪңпјӣ
  • е…Ғи®ёеҜ№иұЎвҖңиҮӘж•‘вҖқпјҢеҚіеңЁ finalize() дёӯйҮҚж–°и®©иҮӘе·ұеҸҳдёәеҸҜиҫҫеҜ№иұЎпјҲжҜ”еҰӮжҠҠ this иөӢеҖјз»ҷжҹҗдёӘйқҷжҖҒеҸҳйҮҸпјүгҖӮ
  • GC еҲӨе®ҡеҜ№иұЎвҖңдёҚеҸҜиҫҫвҖқеҗҺпјҢиӢҘйҮҚеҶҷдәҶ finalize()пјҡ
    • дјҡе°ҶиҜҘеҜ№иұЎж”ҫе…Ҙ F-QueueпјҲFinalizer Reference Queueпјүпјӣ
    • з”ұдёҖдёӘ JVM еҶ…йғЁзәҝзЁӢпјҲFinalizer ThreadпјүејӮжӯҘи°ғз”Ёе…¶ finalize() ж–№жі•пјӣ
    • еҰӮжһңеңЁж–№жі•дёӯйҮҚж–°е»әз«Ӣеј•з”Ёй“ҫпјҢGC дјҡвҖңж”ҫдҪ дёҖй©¬вҖқпјӣ
    • жҜҸдёӘеҜ№иұЎзҡ„ finalize() жңҖеӨҡеҸӘдјҡиў«и°ғз”ЁдёҖж¬ЎгҖӮ

5. д»Җд№Ҳжғ…еҶөдёӢдјҡдә§з”ҹStackOverflowErrorпјҲж ҲжәўеҮәпјүе’ҢOutOfMemoryErrorпјҲе ҶжәўеҮәпјүпјҹжҖҺд№ҲжҺ’жҹҘпјҹ

StackOverflowErrorпјҲж ҲжәўеҮәпјү

еёёи§ҒеҺҹеӣ 

  1. йҖ’еҪ’и°ғз”ЁиҝҮж·ұпјҲжңҖеёёи§ҒпјүгҖӮ

  2. ж–№жі•и°ғз”ЁеұӮзә§иҝҮж·ұпјҢжҜҸдёӘж–№жі•еҚ з”Ёж Ҳеё§иҝҮеӨҡпјӣ

  3. и®ҫзҪ®зҡ„ ж ҲеӨ§е°ҸиҝҮе°ҸпјҲ-Xss еҸӮж•°пјүпјӣ

жҺ’жҹҘ

жҹҘзңӢејӮеёёе Ҷж ҲдҝЎжҒҜпјҡеҮҪж•°и°ғз”ЁжҳҜеҗҰжӯ»йҖ’еҪ’пјӣ

и°ғиҜ•з»ҲжӯўжқЎд»¶пјҡзЎ®и®ӨйҖ’еҪ’жңүжҳҺзЎ®иҝ”еӣһжқЎд»¶пјӣ

и°ғж•ҙ JVM еҸӮж•°пјҲеҰӮеўһеӨ§зәҝзЁӢж ҲеӨ§е°Ҹпјүпјҡ-Xss512kпјҲй»ҳи®Өпјү вҶ’ -Xss1m

OutOfMemoryErrorпјҲе ҶжәўеҮәпјү

еёёи§ҒеҺҹеӣ 

  1. еҲӣе»әдәҶиҝҮеӨҡеҜ№иұЎпјҢжңӘйҮҠж”ҫеј•з”Ёпјӣ

  2. еӨ§йӣҶеҗҲжҲ–зј“еӯҳжңӘжё…зҗҶпјҲеҰӮ List дёҚж–ӯж·»еҠ е…ғзҙ пјүпјӣ

  3. еҶ…еӯҳжі„жјҸпјҡеҜ№иұЎеә”иҜҘиў«еӣһж”¶еҚҙиў«ж„ҸеӨ–еј•з”Ёпјӣ

  4. JVM е ҶеҶ…еӯҳи®ҫзҪ®еӨӘе°ҸпјҲй»ҳи®ӨжңҖеӨ§еҶ…еӯҳдёҚи¶іпјүпјӣ

  5. зәҝзЁӢж•°иҝҮеӨҡй—ҙжҺҘиҖ—е°Ҫе Ҷз©әй—ҙпјӣ

  6. жҹҗдәӣеңәжҷҜдёӢ String.intern() еҜјиҮҙе…ғз©әй—ҙзҲҶж»ЎпјҲиҖҒзүҲжң¬ PermGenпјүпјӣ

жҺ’жҹҘж–№жі•

  • жҹҘзңӢ GC ж—Ҙеҝ—/еҶ…еӯҳзӣ‘жҺ§пјҡ

    • дҪҝз”Ё -XX:+PrintGCDetails -Xloggc:gc.logпјӣ
    • жҲ– JVisualVMгҖҒMATгҖҒJProfiler е·Ҙе…·пјӣ

  • е®ҡдҪҚеҶ…еӯҳжі„жјҸеҜ№иұЎпјҡ

    • дҪҝз”Ё MAT е·Ҙе…·еҲҶжһҗ .hprof ж–Үд»¶пјӣ
    • жҹҘжүҫ вҖңGC RootsвҖқ ејәеј•з”Ёй“ҫпјӣ

  • и°ғж•ҙ JVM еҸӮж•°пјҲеҰӮеўһеӨ§е ҶеҶ…еӯҳпјүпјҡ-Xms512m -Xmx1024m

6. д»Җд№ҲжҳҜзәҝзЁӢжұ пјҢжңүд»Җд№Ҳзұ»еһӢпјҢеҰӮдҪ•еҲӣе»әпјҹ

д»Җд№ҲжҳҜзәҝзЁӢжұ пјҹ

зәҝзЁӢжұ пјҲThread PoolпјүжҳҜдёҖз§ҚеӨҚз”ЁзәҝзЁӢиө„жәҗзҡ„жңәеҲ¶пјҢз”ЁдәҺйҷҗеҲ¶е’Ңз®ЎзҗҶзәҝзЁӢзҡ„еҲӣе»әдёҺй”ҖжҜҒпјҢйҒҝе…Қйў‘з№ҒеҲӣе»ә/й”ҖжҜҒзәҝзЁӢеёҰжқҘзҡ„жҖ§иғҪжҚҹиҖ—гҖӮ

жң¬иҙЁдёҠе°ұжҳҜдёҖдёӘзәҝзЁӢзҡ„е®№еҷЁпјҢеҸҜд»ҘеҸҚеӨҚдҪҝз”Ёе·ІжңүзәҝзЁӢжү§иЎҢд»»еҠЎгҖӮ

зәҝзЁӢжұ зҡ„дҪңз”Ёпјҹ

дјҳзӮ№иҜҙжҳҺ
йҷҚдҪҺиө„жәҗејҖй”ҖеӨҚз”Ёе·ІжңүзәҝзЁӢпјҢйҒҝе…Қйў‘з№ҒеҲӣе»әй”ҖжҜҒзәҝзЁӢ
жҸҗй«ҳе“Қеә”йҖҹеәҰжҸҗдәӨд»»еҠЎеҚіеҸҜз«ӢеҚіжү§иЎҢпјҢдёҚйңҖзӯүеҫ…ж–°зәҝзЁӢеҲӣе»ә
жҸҗдҫӣзәҝзЁӢз®ЎзҗҶжңәеҲ¶жҺ§еҲ¶жңҖеӨ§е№¶еҸ‘ж•°гҖҒи¶…ж—¶зӯ–з•ҘгҖҒд»»еҠЎжҺ’йҳҹзӯ–з•Ҙзӯү
жҸҗй«ҳзі»з»ҹзЁіе®ҡжҖ§йҳІжӯўж— йҷҗеҲ¶еҲӣе»әзәҝзЁӢеҜјиҮҙ OOM жҲ–зі»з»ҹеҙ©жәғ

зәҝзЁӢжұ зҡ„зұ»еһӢ

зәҝзЁӢжұ зұ»еһӢеҲӣе»әж–№ејҸзү№зӮ№иҜҙжҳҺ
еӣәе®ҡеӨ§е°ҸзәҝзЁӢжұ Executors.newFixedThreadPool(n)зәҝзЁӢж•°еӣәе®ҡпјҢйҖӮеҗҲиҙҹиҪҪзЁіе®ҡзҡ„еңәжҷҜпјӣи¶…еҮәйғЁеҲҶжҺ’йҳҹжү§иЎҢ
еҚ•зәҝзЁӢзәҝзЁӢжұ Executors.newSingleThreadExecutor()еҸӘжңүдёҖдёӘзәҝзЁӢпјҢйЎәеәҸжү§иЎҢд»»еҠЎFIFOпјҢйҖӮз”ЁдәҺйЎәеәҸд»»еҠЎеӨ„зҗҶ
зј“еӯҳзәҝзЁӢжұ пјҲж— йҷҗеӨ§пјүExecutors.newCachedThreadPool()з©әй—ІзәҝзЁӢеҸҜеӨҚз”ЁпјҢзәҝзЁӢеҸҜж— йҷҗеўһй•ҝпјҲInteger.MAX_VALUEпјүпјҢйҖӮеҗҲд»»еҠЎеҜҶйӣҶдҪҶжү§иЎҢеҝ«зҡ„еңәжҷҜ
е®ҡж—¶д»»еҠЎзәҝзЁӢжұ Executors.newScheduledThreadPool(n)ж”ҜжҢҒе®ҡж—¶д»»еҠЎжҲ–е‘Ёжңҹд»»еҠЎжү§иЎҢ
е·ҘдҪңзӘғеҸ–зәҝзЁӢжұ пјҲJava 8+пјүExecutors.newWorkStealingPool()еҲ©з”ЁеӨҡж ё CPUпјҢиҮӘеҠЁиҝӣиЎҢд»»еҠЎеҲҶй…ҚдёҺиҙҹиҪҪеқҮиЎЎ

иҮӘе®ҡд№үзәҝзЁӢжұ 

йҖҡиҝҮ ThreadPoolExecutor зұ»жһ„е»әпјҢжӣҙзҒөжҙ»пјҡ

ThreadPoolExecutor executor = new ThreadPoolExecutor(
    corePoolSize,         // ж ёеҝғзәҝзЁӢж•°
    maximumPoolSize,      // жңҖеӨ§зәҝзЁӢж•°
    keepAliveTime,        // з©әй—ІзәҝзЁӢеӯҳжҙ»ж—¶й—ҙ
    TimeUnit.SECONDS,     // ж—¶й—ҙеҚ•дҪҚ
    new LinkedBlockingQueue<>(100), // йҳ»еЎһйҳҹеҲ—пјҲд»»еҠЎзӯүеҫ…еҢәпјү
    Executors.defaultThreadFactory(),  // зәҝзЁӢе·ҘеҺӮ
    new ThreadPoolExecutor.AbortPolicy() // жӢ’з»қзӯ–з•Ҙ
);

жӢ’з»қзӯ–з•ҘиҜҙжҳҺпјҡ

зӯ–з•ҘеҗҚиҜҙжҳҺ
AbortPolicyпјҲй»ҳи®ӨпјүжҠӣејӮеёёжӢ’з»қд»»еҠЎ
CallerRunsPolicyз”ұи°ғз”ЁиҖ…зәҝзЁӢжү§иЎҢд»»еҠЎпјҲйҷҚзә§пјү
DiscardPolicyзӣҙжҺҘдёўејғд»»еҠЎ
DiscardOldestPolicyдёўејғйҳҹеҲ—дёӯжңҖиҖҒзҡ„д»»еҠЎпјҢе°қиҜ•жҸҗдәӨж–°д»»еҠЎ

жіЁж„ҸдәӢйЎ№

  • дёҚиҰҒзӣҙжҺҘдҪҝз”Ё Executors еҲӣе»әзәҝзЁӢжұ пјҲеҺҹеӣ пјҡй»ҳи®Өзӯ–з•ҘдёҚе®үе…ЁпјҢжҳ“OOMпјүпјҢжҺЁиҚҗз”Ё ThreadPoolExecutor жҳҺзЎ®еҸӮж•°пјӣ

  • еҜ№дәҺ Web жңҚеҠЎзұ»зі»з»ҹпјҢе»әи®®ж №жҚ® CPUж ёеҝғж•° й…ҚзҪ®зәҝзЁӢж•°пјҲеҰӮ CPUж ёж•° * 2пјүпјӣ

  • й…ҚеҗҲзӣ‘жҺ§е·Ҙе…·пјҲеҰӮ JVisualVMпјүи§ӮеҜҹзәҝзЁӢжұ зҠ¶жҖҒпјҢжңүеҠ©дәҺи°ғдјҳгҖӮ

7. дёәд»Җд№ҲиҰҒдҪҝз”ЁзәҝзЁӢ

еҺҹеӣ и§ЈйҮҠ
1. йҷҚдҪҺиө„жәҗејҖй”ҖзәҝзЁӢеӨҚз”ЁпјҢйҒҝе…Қйў‘з№ҒеҲӣе»әдёҺй”ҖжҜҒзәҝзЁӢеёҰжқҘзҡ„жҖ§иғҪејҖй”ҖгҖӮ
2. жҸҗй«ҳе“Қеә”йҖҹеәҰд»»еҠЎжҸҗдәӨеҗҺеҸҜд»ҘзӣҙжҺҘеӨҚз”Ёе·ІжңүзәҝзЁӢпјҢз«ӢеҚіжү§иЎҢпјҢж— йңҖзӯүеҫ…зәҝзЁӢеҲӣе»әгҖӮ
3. жҺ§еҲ¶жңҖеӨ§е№¶еҸ‘йҮҸйҷҗеҲ¶еҗҢж—¶иҝҗиЎҢзҡ„зәҝзЁӢж•°пјҢйҳІжӯўзі»з»ҹиө„жәҗиҖ—е°ҪеҜјиҮҙеҙ©жәғгҖӮ
4. е®һзҺ°д»»еҠЎжҺ’йҳҹдёҺи°ғеәҰеҶ…йғЁз»ҙжҠӨд»»еҠЎйҳҹеҲ—пјҢж”ҜжҢҒд»»еҠЎжҺ’йҳҹгҖҒе»¶иҝҹгҖҒе®ҡж—¶гҖҒе‘Ёжңҹжү§иЎҢгҖӮ
5. жҸҗеҚҮзі»з»ҹзЁіе®ҡжҖ§е’ҢеҸҜз»ҙжҠӨжҖ§йҖҡиҝҮз»ҹдёҖзҡ„зәҝзЁӢз®ЎзҗҶжңәеҲ¶пјҢжӣҙжҳ“дәҺзӣ‘жҺ§гҖҒи°ғдјҳе’Ңж•…йҡңжҒўеӨҚгҖӮ
